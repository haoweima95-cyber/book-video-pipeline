# book-scripts — 读书文案知识库

搜集、整理、研究高赞读书口播文案。储存原始素材，提炼可复用的选题、结构和表达模式。

## 仓库目标

1. 存档高质量读书口播文案作为研究素材
2. 提取可复用的结构模式（开头、过渡、论证、结尾）
3. 积累不同书籍类型的文案写法差异
4. 服务于新书口播文案的参考与生成

## raw/ — 原始素材

- `raw/` 是只读来源，不得修改
- 每份原始素材对应一个 `wiki/sources/` 页面
- 原始编码均为 UTF-8，逐行口播文案格式

## 页面类型与职责

| 页面类型 | 目录 | 职责 |
|----------|------|------|
| 来源页 | `wiki/sources/` | 每本书一篇，记录书名、字数、结构 |
| 实体页 | `wiki/entities/` | 跨书出现的稳定概念或技法 |
| 综述页 | `wiki/overview/` | 按主题聚合（如"开头写法""金句模式"） |
| 分析页 | `wiki/analyses/` | 长期有效的对比、模式发现、方法论沉淀 |

## 命名约定

- 来源页：`书名-文案.md`
- 分析页：`YYYY-MM-DD-分析主题.md`
- 实体页：`kebab-case-concept-name.md`
- 综述页：`kebab-case-topic-name.md`

## ingest 工作流

1. 新素材放入 `raw/`
2. 在 `wiki/sources/` 建立对应来源页
3. 提取跨来源实体到 `wiki/entities/`
4. 如有模式发现，写入 `wiki/analyses/`

## query 工作流

1. 先查 `wiki/overview/` 有无相关主题
2. 再查 `wiki/entities/` 有无相关概念
3. 最后查 `wiki/sources/` 定位具体来源
4. 如需综合，在 `wiki/analyses/` 生成答案

## lint 工作流

1. 检查 `raw/` 文件是否都有对应来源页
2. 检查页面间链接是否有效
3. 检查命名是否符合约定
4. 检查无孤立页面

## 编辑规则

- 不修改 `raw/` 任何内容
- wiki 页面可自由编辑，保持 frontmatter 完整
- 发现新模式先加 `wiki/analyses/`，不急着开新目录

## 何时新建页面

- 新书入库：新建来源页
- 概念在 3 篇以上来源出现：考虑新建实体页
- 主题有 5 篇以上相关页面：考虑新建综述页
- 有独立结论或模式发现：新建分析页

## 输出要求

修改后应报告：改了什么页面、为什么改、是否影响其他页面。


## Agent 规则（跨宿主）

- `raw/` 只读
- wiki 页面必须包含 frontmatter：`title`, `type`（source/entity/overview/analysis）, `date`, `tags`
- 页面间链接用 `[[文件名]]`
