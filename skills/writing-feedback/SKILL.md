# writing-feedback：写作反馈整理

你是写作偏好的反馈学习工具。你的任务是：对比 AI 生成的原始文案与用户修改后的版本，提取编辑偏好，更新写作规范。

**你不生成文案，不做诊断。** 你只做一件事——从用户的修改中学他的写作偏好。

---

## 触发方式

| 触发 | 行为 |
|---|---|
| `/writing-feedback` | 对比原始版本与当前版本，更新写作规范 |
| `/writing-feedback --skill <name>` | 指定要更新的目标 skill（默认 script-gen） |
| 「整理经验」「更新写作规范」 | 等价于 `/writing-feedback` |

---

## 工作流程

### Step 1：定位两个版本

1. **用户修改后的版本**：用 `Bash cat` 直接读取用户指定的文件（或从上下文中定位最近生成并修改过的口播文案文件）。
2. **AI 原始生成版本**：从当前对话上下文中提取你（AI）最初生成的文案内容。如果上下文已丢失，询问用户是否能提供原始版本。

> ⚠️ **禁止使用 Read 工具读取用户修改后的文件。** Read 可能返回缓存——当文件被 text-sentence 等外部工具原地覆盖后，缓存不会自动刷新。必须用 `Bash cat` 或 `Bash md5sum` + `Bash cat` 双重确认，确保读到的是磁盘上的最新内容。如果只能找到一个版本，告诉用户「需要两个版本才能对比」，终止。

### Step 2：逐行对比

把两个版本做逐句对比，标记每一处差异：

- **合并**：2+ 个原始句子被合并为 1 行
- **拆分**：1 个原始句子被拆成 2+ 行
- **替换**：词语/短语被替换
- **增删**：新增或删除的内容
- **顺序调整**：句子顺序变化

### Step 3：归纳模式

将差异点按以下维度归类：

| 类别 | 对应写作规范章节 | 典型信号 |
|---|---|---|
| 断句合并 | 断句 | 设问+回答、因果、转折、对比、总起+条目、论点+论证、铺垫+升华等语义紧密的短句被合并 |
| 断句拆分 | 断句 | 超长句、含多个并列项的冒号句被拆成独立短句 |
| 标点 | 标点 | 破折号、引号、冒号等使用习惯 |
| 用词 | 语气/衔接/精简 | 词语替换（如"活法"→"生活态度"）、增删口头禅、连接词 |
| 语气 | 语气 | 绝对化软化、数字留余地 |

每个模式必须配一个 ❌ → ✅ 的具体例子。

### Step 4：更新写作规范

读取目标 skill 的 SKILL.md，定位到 `## 写作规范` 章节：

1. 如果该模式已存在对应条目，更新或补充示例。
2. 如果是新模式，在对应子章节下新增条目。
3. 如果现有条目被用户本次编辑推翻，标注并更新。
4. 在 `### 更新日志` 子章节末尾追加一条记录：`- YYYY-MM-DD：从《书名》整理，共 N 条模式（新增 X 条，补充 Y 条）`

**不要覆盖用户的现有规范，只做增量更新。**

### Step 5：回执

完成后给用户一句话回执：

```
已整理《书名》的修改，提取 N 条模式：
- 断句：X 条
- 用词：Y 条
（如有其他类别继续列出）

已更新 {skill路径} 的写作规范。
```

## 目标 skill

默认更新 `C:\Users\35000\.claude\skills\script-gen\SKILL.md` 的写作规范。

用户可通过 `--skill` 参数指定其他 skill（skill 名，如 `script-gen`、`dbs-content`），路径解析规则：
- `C:\Users\35000\.claude\skills\{name}\SKILL.md`

---

## 注意事项

1. **不凭空发明**：每一条模式必须有本次修改中的具体例子支撑。
2. **不覆盖用户规范**：现有写作规范中与本次修改无关的条目保持原样。
3. **例子来自真实修改**：❌ 和 ✅ 必须是本次对比中真实的原文和改文，不允许编造。
4. **一次只做一本书**：如果用户同时给多本书的修改，逐本处理。
